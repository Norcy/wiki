# 结构型模式
## 适配器模式
作为两个不兼容的接口之间的桥梁

举例：

1. 读卡器是作为内存卡和笔记本之间的适配器
2. 美国电器 110V，中国 220V，就要有一个适配器将 110V 转化为 220V
3. iOS 里的协议
4. 腾讯视频工程的PlayerAdapter，实现对新老播放器的适配（使用组合的方式）



## 外观模式
针对复杂的子系统提供了单一的 API 接口


## 装饰器模式
向一个现有的对象添加新的功能，同时又不改变其结构

举例：iOS 里的分类（Category）和代理（Delegation）

# 行为型模式

## 观察者模式
当对象间存在一对多关系时，则使用观察者模式。当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并被自动更新

举例：

1. NSNotification
2. KVO

## 备忘录模式
在不破坏封装性的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态

为了符合迪米特法则，还要增加一个管理备忘录的类，客户不与备忘录类耦合，与备忘录管理类耦合

所谓的不破坏封装性，就是原对象的私有数据的私有性质保持不变

举例：

1. 游戏存档
2. 浏览器的后退
3. NSUserDefaults
4. Archiving，即 NSCoding 协议（调用者并不需要知道被归档对象的私有数据就可以进行归档，即不影响私有数据的私有性质）


## 命令模式
将一个请求封装为一个对象，并传给调用对象

举例：

1. NSInvocation
2. Target-Action机制
3. 新播放器中的事件传递机制
4. 涂鸦功能的撤销操作，可以将每次绘制操作封装为对象存起来，以便于实现撤销操作
